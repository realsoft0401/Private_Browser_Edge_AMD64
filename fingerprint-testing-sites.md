# 浏览器指纹检测与打分站点整理

这份清单服务 `Private_Browser_Edge_AMD64` 镜像联调、代理链路排障、WebRTC 防漏核验和浏览器运行一致性自测。

它的职责边界是沉淀常用检测站点、适合观察的能力维度和推荐使用场景，方便后续维护 AMD64 浏览器镜像、代理配置和运行时指纹保护时快速复用。

它不负责给出“绕过平台风控”的工程方案，也不把第三方站点评分直接当成业务可用性的唯一判断标准。真正的运行可用性仍要结合 Edge 容器启动原子性、CDP/VNC 健康探测、代理出口校验和上层任务失败事实一起判断。

## 一、综合全维度检测

按专业度和排障价值排序。

### 1. BrowserLeaks

- 网址：[https://browserleaks.com/](https://browserleaks.com/)
- 功能：全维度浏览器指纹检测，覆盖 IP、WebRTC、Canvas、WebGL、UA、字体、插件、时区、DNS、硬件信息，以及部分 TLS/JA3 相关观察项。
- 特点：单项维度拆得最细，能逐项核对篡改是否真实生效；虽然不主打总分，但更适合技术排障和镜像基线比对。
- 适用场景：反复核验代理环境、WebRTC 防漏、显卡/渲染器暴露、字体特征、时区语言一致性，以及 Chromium 启动参数调整后的逐项回归。

### 2. AmIUnique

- 网址：[https://amiunique.org/](https://amiunique.org/)
- 功能：浏览器指纹唯一性评估与统计分析，给出指纹辨识度、相似设备数量和部分熵值拆分。
- 特点：更强调“这份指纹在全网是否过于独特”，适合从隐私和去重概率角度观察风险，不只是看单项字段是否暴露。
- 适用场景：评估当前镜像和运行参数生成的指纹是否过于突出，判断某些字段组合是否导致“过于少见”的唯一性问题。

### 3. Whoer.net

- 网址：[https://whoer.net/](https://whoer.net/)
- 功能：匿名度评分、代理识别、DNS 泄露、WebRTC 泄露、时区和语言匹配度检查。
- 特点：首页分数直观，适合快速做一轮总览；同时兼顾代理/IP 侧观测，日常回归成本低。
- 适用场景：快速打分、首轮检查代理和地区一致性、联调结束后的简化验收。

## 二、专项指纹检测

这组站点更适合在综合检测之后，用来复核具体单项。

### 1. FingerprintJS Demo

- 网址：[https://fingerprintjs.github.io/fingerprintjs/](https://fingerprintjs.github.io/fingerprintjs/)
- 功能：生成浏览器唯一指纹哈希，观察刷新后哈希是否稳定或随机变化。
- 特点：背后是行业常见的指纹库 Demo，便于验证指纹是否被意外固化，或随机策略是否真正生效。
- 适用场景：比较不同启动参数、不同 profile、不同代理模式下的哈希变化，确认镜像调整没有引入异常固化。

### 2. WebGL Report

- 网址：[https://webglreport.com/](https://webglreport.com/)
- 功能：查看 WebGL 厂商、渲染器、图形能力和显卡相关暴露信息。
- 特点：聚焦图形栈，适合排查容器内软件渲染、显卡暴露异常或 WebGL 指纹不符合预期的问题。
- 适用场景：检查 `renderer/vendor` 是否和预期一致，确认 Chromium、Xvfb、容器图形栈没有出现异常特征。

### 3. IPLeak WebRTC Test

- 网址：[https://www.ipleak.net/webrtc-test/](https://www.ipleak.net/webrtc-test/)
- 功能：专项检查 WebRTC 是否泄露真实内网 IP 或公网 IP。
- 特点：页面聚焦单项风险，适合确认 WebRTC 保护参数、iptables 规则和代理链路是否真的挡住了非代理 UDP 暴露。
- 适用场景：每次调整 WebRTC、Mihomo、TUN、UDP 限制或代理模式后，做专项回归。

### 4. JA3er

- 网址：[https://ja3er.com/](https://ja3er.com/)
- 功能：观察 TLS 握手指纹和 JA3 哈希。
- 特点：属于协议层高阶观测，更适合用来确认底层 TLS 指纹是否明显偏离常规浏览器。
- 适用场景：高阶风控排障、协议层一致性观察，以及代理/容器环境变化后的 TLS 特征比对。

## 三、隐私 / 反爬风控观察类

这类站点更偏轻量自测、隐私提示或网络泄露观察，适合作为补充视角。

### 1. Cover Your Tracks

- 网址：[https://coveryourtracks.eff.org/](https://coveryourtracks.eff.org/)
- 功能：评估浏览器被追踪的难易程度，并给出通俗化的隐私提示。
- 特点：EFF 出品，偏公益和用户教育导向，解释成本低，适合快速理解“为什么这个环境容易被区分”。
- 适用场景：做隐私风险初筛、给非底层同学解释指纹唯一性问题，或补充观察抗追踪能力。

### 2. IPLeak

- 网址：[https://ipleak.net/](https://ipleak.net/)
- 功能：综合检查 IP、DNS、WebRTC、地理位置和时区等网络侧暴露信息。
- 特点：页面简洁，网络泄露项集中，适合确认代理出口和本地泄露是否一致。
- 适用场景：快速确认 DNS、WebRTC、时区和出口地域是否匹配，作为代理配置调整后的基础验收。

## 四、推荐使用顺序

### 1. 快速总览

- 先看 `Whoer.net`，确认匿名度评分、代理状态、时区和语言是否大体一致。

### 2. 唯一性判断

- 再看 `AmIUnique`，判断当前指纹是否过于独特，哪些字段熵值偏高。

### 3. 深度逐项排错

- 用 `BrowserLeaks` 逐项核验 WebRTC、Canvas、WebGL、字体、插件、DNS、时区等细项。

### 4. 单项复核

- 哈希稳定性看 `FingerprintJS Demo`。
- 图形栈看 `WebGL Report`。
- WebRTC 泄露看 `IPLeak WebRTC Test`。
- TLS/JA3 看 `JA3er`。

## 五、使用提醒

- 大部分境外检测站点更适合在代理出口稳定、时区语言已对齐的前提下使用，否则评分容易被地区和网络抖动干扰。
- 不要把第三方站点分数直接当成“平台业务一定可用”的结论，必须结合 Edge 运行日志、CDP 健康探测、代理出口事实和上层任务结果一起看。
- 不建议在短时间内高频反复刷新同一检测页，部分站点会记录访问行为，过度测试会影响观察结果。
- 如果一次调整同时改了代理、时区、语言、WebRTC 和启动参数，建议分批验证，否则很难判断到底是哪一项引入了异常。
